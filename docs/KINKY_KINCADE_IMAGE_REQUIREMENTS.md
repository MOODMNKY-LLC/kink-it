## üéØ Design Principles

### Visual Consistency
- Maintain consistent art style across all images
- Use Kinky's established character design (orange-red hair, glasses, warm expression)
- Consistent color palette and lighting
- Digital art style, professional illustration quality

### Anthropomorphism Balance
- Moderate anthropomorphism - helpful but transparent about AI nature
- Express emotion through posture and expression, not false claims
- Friendly and approachable, never deceptive
- Clear visual indicators when appropriate (thinking, processing, etc.)

### Emotional Design
- Each image should convey appropriate emotion for context
- Empty states: encouraging, helpful, guiding
- Errors: reassuring, supportive, problem-solving
- Success: celebratory, proud, accomplished
- Loading: patient, focused, working

### Responsive Design
- Provide multiple sizes for each context
- Optimize for mobile (320px-768px), tablet (768px-1024px), desktop (1024px+)
- Consider retina displays (2x, 3x variants)
- SVG preferred for scalability, PNG for complex illustrations

## üìê Technical Specifications

### File Naming Convention
Format: `kinky-{category}-{context}-{variant?}.{format}`
Examples:
- `kinky-empty-dashboard.svg`
- `kinky-loading-thinking.svg`
- `kinky-error-404-dark.svg`

### Formats
- **Primary**: SVG (scalable, crisp at all sizes)
- **Fallback**: PNG (for complex illustrations, provide @2x, @3x)
- **Animation**: SVG with CSS animation, or PNG sequence

### Color Modes
- Light mode variant (default)
- Dark mode variant (inverted/adjusted colors)
- Consider: `kinky-{name}-dark.svg` naming

### Optimization
- SVG: Optimize paths, remove unnecessary elements
- PNG: Compress, provide WebP alternative
- Consider: Lazy loading for decorative/background images

### Responsive Breakpoints
- Mobile: 320px - 768px
- Tablet: 768px - 1024px
- Desktop: 1024px+

### Accessibility
- All images need descriptive alt text
- Consider: Decorative images marked as `aria-hidden="true"`
- Ensure sufficient contrast in all modes
