## Introduction

The Tasks module allows Dominants to assign tasks to Submissives, track completion, and manage accountability. Tasks can be routine daily activities, one-time assignments, protocol-linked requirements, or low-energy mode adaptations. This module provides structure and clear expectations while respecting submission states.

---

## What Are Tasks?

**Tasks** are specific assignments given by Dominants to Submissives. They can be:
- **Routine Tasks**: Daily or regular assignments (morning greeting, coffee service)
- **One-Time Tasks**: Specific assignments (weekly reflection, meal planning)
- **Protocol-Linked Tasks**: Tasks tied to specific rules or protocols
- **Low-Energy Mode Tasks**: Simplified tasks for low-energy submission states

## Creating Tasks

### For Dominants

#### Step 1: Define the Task

Consider:
- **What**: What needs to be done?
- **Why**: Why does it matter?
- **How**: How should it be completed?
- **When**: When is it due?

#### Step 2: Set Task Details

**Title**: Clear, concise name
- "Morning Greeting"
- "Coffee Service"
- "Weekly Reflection"

**Description**: Detailed instructions
- What's expected
- How to complete it
- Acceptance criteria
- Why it matters

**Priority**: Importance level
- Low: Nice to have
- Medium: Standard expectation
- High: Important, prioritize
- Urgent: Critical, immediate

**Point Value**: Reward points for completion
- Routine tasks: 10-15 points
- Special tasks: 20-25 points
- Low-energy tasks: 5 points

**Due Date**: When it's due
- Specific date/time
- Recurring (daily, weekly)
- No due date (flexible)

#### Step 3: Set Proof Requirements

**No Proof**: Self-completed, no verification needed
- Simple rituals
- Basic check-ins
- Trust-based tasks

**Text Proof**: Written verification required
- Reflections
- Acknowledgment messages
- Learning assignments

**Photo Proof**: Visual verification required
- Exercise completion
- Chore completion
- Outfit approval

**Video Proof**: Video demonstration required
- Skill demonstrations
- Protocol practice
- Special assignments

#### Step 4: Assign and Communicate

1. Create task in app
2. Assign to submissive
3. Communicate expectations
4. Answer questions
5. Track completion

### For Submissives

#### Understanding Tasks

When you receive a task:
1. **Read Carefully**: Understand expectations
2. **Check Due Date**: Know when it's due
3. **Note Proof Requirements**: Know what proof is needed
4. **Ask Questions**: Clarify if unclear
5. **Plan Completion**: Schedule time to complete

#### Completing Tasks

1. **Do the Work**: Complete the task
2. **Submit Proof**: If proof required, submit it
3. **Mark Complete**: Update status
4. **Wait for Approval**: If approval needed, wait
5. **Receive Points**: Points awarded on completion

#### Low-Energy Mode

When in low-energy submission state:
- Tasks may be simplified
- Expectations adjusted
- Focus on self-care
- Communicate capacity
- Request modifications if needed

## Task Status

### Status Types

**Pending**: Assigned but not started
- Task created
- Waiting to begin
- Can be started anytime

**In Progress**: Started but not complete
- Work begun
- In process
- Can be updated

**Completed**: Finished, awaiting approval
- Task done
- Proof submitted (if required)
- Waiting for Dominant approval

**Approved**: Completed and approved
- Dominant approved
- Points awarded
- Task closed

**Cancelled**: Task cancelled
- No longer needed
- Points not awarded
- Can be recreated if needed

## Best Practices

### For Dominants

1. **Start Small**: Begin with 2-3 tasks, add gradually
2. **Be Clear**: Specific instructions prevent confusion
3. **Set Realistic Expectations**: Achievable tasks build success
4. **Acknowledge Completion**: Recognize effort and completion
5. **Adjust for State**: Modify tasks for low-energy mode
6. **Review Regularly**: Check what's working, adjust as needed

### For Submissives

1. **Read Instructions**: Understand expectations fully
2. **Ask Questions**: Clarify if unclear
3. **Submit Proof Promptly**: Don't delay proof submission
4. **Communicate Issues**: If struggling, say so
5. **Request Adjustments**: If task needs modification, ask
6. **Celebrate Completion**: Acknowledge your effort

### Task Design Principles

1. **Clear**: Unambiguous instructions
2. **Achievable**: Realistic expectations
3. **Meaningful**: Purposeful, not busywork
4. **Flexible**: Adaptable when needed
5. **Trackable**: Can verify completion

## Low-Energy Mode Tasks

### Adapting Tasks

When submissive is in low-energy state:

1. **Simplify**: Reduce complexity
2. **Reduce Quantity**: Fewer tasks
3. **Lower Expectations**: Basic completion okay
4. **Focus on Self-Care**: Prioritize wellbeing
5. **Communicate**: Discuss capacity

### Example Adaptations

**Normal Task**: "Complete 30-minute exercise routine with photo proof"

**Low-Energy Adaptation**: "Just check in with me about how you're feeling. No pressure, just want to know you're okay."

## Troubleshooting

### "Too Many Tasks"

**Problem**: Overwhelmed by number of tasks.

**Solution**:
- Reduce number of active tasks
- Prioritize what matters most
- Combine similar tasks
- Adjust for submission state

### "Tasks Feel Like Chores"

**Problem**: Tasks feel like work rather than service.

**Solution**:
- Review "why it matters" for each task
- Focus on meaningful tasks
- Reduce routine tasks
- Add variety

### "Can't Complete Tasks"

**Problem**: Struggling to complete assigned tasks.

**Solution**:
- Communicate capacity honestly
- Request adjustments
- Discuss what's challenging
- Adjust expectations together
